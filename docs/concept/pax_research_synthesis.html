Key research findings — what changes in our design

SELF-REFINEvalidates our loop
Formal proof that LLMs iteratively self-correct without retraining — generate → evaluate → refine. Works on code, math, creative tasks. Stronger models (GPT-4 class) benefit more. This is the academic foundation for our visual correction loop.
→ The vision loop isn't just a nice idea. It's proven. We should call it explicitly in our docs and implement it as a first-class workflow, not a footnote.
Madaan et al., NeurIPS 2023 · confirmed by R3V (NAACL 2025)
GameTileNetdirect asset
2,142 semantically-labeled pixel art tiles from OpenGameArt.org (CC licensed), with affordance types, connectivity, object names. 361 normalized tags. Achieves 92.2% classification accuracy with ResNet18. This is exactly the stamp corpus foundation we need.
→ Don't build the semantic tag vocabulary from scratch. License-compatible tiles from GameTileNet seed the stamp corpus. PAX ships with a real dataset on day one.
Chen & Jhala, AAAI AIIDE 2025 · arxiv 2507.02941
Narrative-to-Scene pipelinenew direction
LLM extracts Object-Relation-Object triples from narrative → semantic embedding retrieval from GameTileNet → Cellular Automata terrain → placed objects. Works across 10 diverse stories. Key gap identified: no spatial conflict resolution.
→ PAX can absorb this entire pipeline. "I want a forest village with a tavern near water" → LLM extracts predicates → PAX's semantic WFC places tiles. Natural language to playable map. Nobody has closed this loop properly yet.
arxiv 2509.04481, Aug 2025
Aseprite MCP experimentconfirms gap we fill
Claude Opus 4 can produce creative pixel art via MCP but per-pixel tool calls are the wrong abstraction. "Drawing pixel art might not be the best use-case for a tool-calling LLM." The researcher's suggested better use-cases: palette recoloring, spritesheet export, dimension correction — all structural tasks.
→ PAX's tile-level abstraction (not pixel-level) is exactly the right granularity. We're solving the problem the researcher identified as unsolved.
ljvmiranda921.github.io, Jul 2025
Tiled TMX/JSON — universal interchangeformat decision
Tiled's TMX (XML) and JSON formats are natively supported by Godot 4 (YATI plugin), Unity (SuperTiled2Unity), and 40+ other engines. TexturePacker JSON Hash is the de-facto sprite atlas standard for 48+ engines including Unity, UE, Godot, libGDX, Phaser, Cocos2d. Both formats are trivial to generate.
→ PAX doesn't need custom Unity/Godot exporters. Export Tiled JSON + TexturePacker JSON Hash = works in every engine. Two formats, universal reach.
doc.mapeditor.org · codeandweb.com · github YATI/SuperTiled2Unity
OpenGameArt CC0 on HuggingFacefree training data
Complete CC0 dataset of OpenGameArt.org assets with metadata as JSONL on HuggingFace (nyuuzyou/OpenGameArt-CC0). Includes 2D art, textures, tilesets. No attribution required (CC0 = public domain). Combined with GameTileNet annotations, this is the stamp corpus.
→ Pre-ship PAX with a curated CC0 stamp library — real art, not placeholder. Makes day-one generation meaningfully better and gives the community a starting point.
huggingface.co/nyuuzyou/OpenGameArt-CC0

The SELF-REFINE visual correction loop — how it works in PAX

① GENERATE
LLM writes PAX
Tile source from prompt + theme + palette constraints
→
② RENDER
Go renders PNG
16× zoom, returned as base64 in MCP response
→
③ VALIDATE
Tool checks rules
Edges, palette, size, WFC compat — structured errors
→
④ SEE + CRITIQUE
LLM examines
Vision: "edge row 8 has a gap" + validation errors
→
⑤ REFINE
Targeted fix
Delta edit — only broken rows/regions rewritten
✓ accept
Why this works better than one-shot generation: SELF-REFINE literature shows 10–20% quality improvement per iteration on structured output tasks. For pixel art specifically, the LLM can literally see the rendered result — not just textual error messages — and reason about what looks wrong visually. This is the difference between "row 8 col 3 is wrong" and "I can see the left edge doesn't align." Max 3 refinement passes before escalating to user. Passes 2+ only fix the delta, not the full tile.

Six concrete improvements from the research

1. Absorb GameTileNet as the stamp corpus seed
2,142 CC-licensed, semantically-annotated tiles ready to ship. Convert to PAX format at tool build time. Gives every PAX install a real stamp library — not placeholder art — on day one. The affordance labels map directly onto semantic tile DNA tags.
GameTileNet AAAI 2025 · OpenGameArt CC0 HuggingFace dataset
2. Narrative-to-map: "a rainy fishing village" → tilemap
PAX + LLM can implement the full Narrative-to-Scene pipeline. LLM extracts spatial predicates from natural language, PAX's semantic WFC resolves them into a valid tilemap. This is the killer demo: type a description, get a playable map. The gap paper identified (spatial conflict resolution) is exactly what semantic WFC rules fix.
arxiv 2509.04481 Narrative-to-Scene pipeline
3. SELF-REFINE: formalize the loop, cap at 3 iterations
The academic literature shows 3 refinement passes capture most improvement. Beyond 3, gains are marginal vs. cost. Implement as: generate → render+validate → LLM critiques via vision → delta refine → re-render → LLM confirms. Stop at accept or 3 passes. Return best result with iteration count in metadata.
Madaan et al. SELF-REFINE NeurIPS 2023
4. GBStudio export: the retro niche nobody serves
GBStudio is huge in the indie scene for Game Boy-style games. It needs a very specific 160×144 background tileset format in a fixed grid PNG. PAX can generate this exactly — and nobody else automates it. A `pax export --format gbstudio` command would make PIXL Studio the only AI tool that outputs GBStudio-ready assets.
spritefusion.com · GBStudio documentation
5. TexturePacker JSON Hash as the primary atlas format
Every engine that doesn't natively read Tiled maps reads TexturePacker JSON. It's the lowest common denominator for sprite atlases. The format is trivial to generate: frames dict, frame rect, trimmed bool, pivot. Make this the default `pax atlas` output, with Tiled JSON as secondary. Both in the same command.
TexturePacker docs · 48+ engine support confirmed
6. Affordance-aware semantic WFC — smarter than edge-only
GameTileNet's affordance taxonomy (walkable, obstacle, collectible, hazard, decoration, character) maps perfectly onto semantic WFC rules. A "hazard" tile cannot be placed adjacent to "spawn" without a "warning" transition. This produces game-sensible maps, not just geometrically valid ones. The WFC propagator gets a 10-line semantic predicate filter.
GameTileNet affordance labels · VGDL taxonomy
