Format Layer Stack (LLMs work top-down, renderer bottom-up)
THEME
@theme / @palette
CSS-like style inheritance. Semantic color roles, scale, dither, validation rules. The LLM never touches hex codes.

LAYER 4
Map / @map
Assembled from tile groups. Can be hand-authored or WFC-generated with seed for reproducibility.
@generator wfc { seed=42 }
LAYER 3
Tile Group / @tilegroup
Complete autotile set (47 topological variants). WFC adjacency rules derived automatically from tile names (N/S/E/W).
dungeon_wall, cave_floor…
LAYER 2
Tile / Sprite / @tile @sprite
Primary authoring unit. 8×8 to 64×64. Composed from micro-tiles OR direct symbols + RLE. Edge declarations for WFC. @diff for animation frames.
wall_NS, hero_walk[4 frames]
LAYER 1
Micro-tile / @micro
Named 4×4 reusable fragments. The BPE tokens of PIXL — frequent visual patterns get semantic names. LLM composes from vocabulary.
corner_nw, wall_cap, crack_v…
LAYER 0
Symbol / Pixel
Single printable char = single palette entry. Max 62 symbols (a-z, A-Z, 0-9). '.' always = transparent. Chars chosen for visual mnemonics.
# = stone, ~ = water, o = gold
Research Foundations

BPE Image Tokenization

Applying Byte-Pair Encoding to visual patches creates shorter, more learnable sequences. PIXL's micro-tile vocabulary is a human-readable implementation of this insight.

Zhang et al. ICLR 2025 · Elsner et al. NeurIPS 2024

Wave Function Collapse

Constraint propagation via entropy minimization. PIXL uses WFC as a validator and completer — the LLM designs tiles, WFC checks and fills gaps.

Gumin 2016 · N-WFC arxiv 2023

LLM Spatial Reasoning Gap

LLMs fail at pixel-space spatial reasoning but excel at semantic-space composition. PIXL's symbolic palette maps characters to meanings, not colors.

SkyPilot blog 2024 · ljvmiranda 2025

Grammar-Based PCG

Generative grammars are the gold standard for cross-asset stylistic coherence in games (Caves of Qud, Townscaper). PIXL's @theme is this grammar.

PCG Survey AAAI 2024
PIXL Source — What the LLM Actually Writes
@theme dark_fantasy { @palette dungeon_dark   // symbol → semantic role → hex @scale 2               // 2× pixel scale on render --stone: '#' #2a1f3d // LLM writes '#', never the hex --lit: '+' lighten(stone, 40%) --water: '~' #1a3a5c --gold: 'o' #c8a035 } @tilegroup dungeon_wall { @wfc auto // edge rules derived from variant names below tile NSEW { // surrounded on all sides grid { 16# / 1#14+1# / 1#6+2s6+1# / 16# } } tile NS { @compose { [wall_cap] [wall_cap] / [wall_side] [wall_side] … } } tile N { @symmetry horizontal; grid { … } } } @sprite hero_walk 16x32 { @frames 4; @fps 8; @loop cycle frame 1 { grid { … full sprite … } } frame 2 { @diff from=1 { 20:7=+, 22:6=# } } // only changed pixels }
CLI Toolchain (Go or Rust)

→ PNG. Reads .pixl, renders pixel-perfect output
pixl validate
→ edge compat, palette, size checks. CI-ready
pixl wfc
→ generate tilemap from tilegroup + constraints
pixl pack
→ sprite atlas PNG + Tiled/Godot JSON
pixl serve
→ HTTP API for web integrations
pixl mcp
→ MCP server (stdio). Full LLM integration
MCP Tools — The LLM Integration Layer
pixl.get_palette(theme)
Returns symbol table + semantic roles. LLM reads this before generating any tile.
pixl.validate_tile(source)
Returns edge errors, unknown symbols, size issues. LLM iterates until clean.
pixl.check_tileable(a, b, dir)
Returns compatible:bool + reason. LLM validates edge matching before committing.
pixl.render_tile(source)
Returns png_base64. LLM can visually inspect what it wrote.
pixl.generate_wfc_map(group, w, h, seed)
Returns complete @map block. LLM can inspect, edit, or accept as-is.
pixl.suggest_compatible(edge_N, edge_W)
Returns tiles that can go here. LLM uses when hand-authoring a map.
Why This Is Groundbreaking
First LLM-native visual authorship formatNot "describe → AI generates" — the LLM IS the author, writing auditable, diffable, editable source.
Semantic compression = natural LLM token spaceMicro-tiles are human-readable BPE. LLMs compose from vocabulary, not coordinate-by-coordinate.
Validation loops = autonomous iterationwrite → validate → fix → validate. Same loop as Claude Code for software, now for art.
WFC as validator, not generatorLLM keeps authorial intent. WFC enforces tileability. No contradiction between design and constraint.
Style coherence by constructionShared @theme means all assets are palette-compatible. Impossible to accidentally use wrong colors.
Git-native art pipelineDiff, review, CI/CD for sprites. Art assets become first-class citizens in version control.
