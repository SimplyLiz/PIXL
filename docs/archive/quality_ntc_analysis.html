The four real blockers and fixes

1. Palette discipline — easy fix
GBA sprites use exactly 16 colors per palette. The LLM needs a hard constraint, not a suggestion.
→ Add @constraint max_palette_size = 16 to theme. Hard validation error on generation if exceeded. LLM gets corrective feedback immediately.
2. Consistent light source — medium fix
Minish Cap uses top-left lighting universally. Every tile, every sprite. Without this constraint the generated tileset feels incoherent even if individual tiles look good.
→ Add @light_source top-left to theme. Inject a 4×4 reference quad into every generation prompt showing shadow direction. Validate: shadow pixels must appear bottom-right of any solid form.
3. Character animation volume — medium fix
~300 frames sounds impossible. But Minish Cap Link really only has ~20 truly distinct pose silhouettes. Walk-L and Walk-R are the same pose mirrored. Frame 3 of a walk cycle is a delta from frame 1.
→ Design 20 canonical skeleton poses. Human artist refines those 20. Everything else: PAX delta system generates 8-frame variations automatically from the canonical set. Sprite animation goes from "impossible" to "two weeks of human polish."
4. World-level visual variation — hardest, needs theme hierarchy
Hyrule Town, Mount Crenel, Castor Wilds, Palace of Winds — each world has a distinct visual language. But Link, UI, and collectibles must look like they belong to the same game across all worlds.
→ PAX needs @theme forest_world extends game_base { ... } — inherited themes. Base theme locks character palette + UI + fonts. World themes only override environment tiles and terrain stamps. LLM generates world art with world theme active; player always uses base theme.
Verdict: A full Zelda: Minish Cap–quality indie game is achievable with PIXL Studio. Environment tilesets — 90% of total visual work — are comfortably in PAX's wheelhouse. Character animation is the honest weak spot; plan 2–3 weeks of human artist time on the 20 canonical poses everything else derives from. Total asset production: from 6–12 months solo artist → 6–8 weeks with PAX + 1 artist doing quality control.

NVIDIA NTC — what it actually is and what it means for PAX

What NTC IS (and why it doesn't apply directly)
NTC compresses PBR texture bundles — albedo, normal, metalness, roughness, AO — for 3D rendering. It trains a tiny neural network per material, stores latents, and decodes individual texels on the GPU in real time.

Reduction: 272 MB → 11 MB (96% compression) on RTX 4090. Requires tensor cores for real-time decode. Still in beta (v0.5).

Why it's wrong for pixel art:
• Pixel art uses indexed palettes — NTC approximates continuous colors
• 16×16 tiles need zero compression — they're already 512 bytes
• NTC requires RTX GPU at runtime — indie games target all hardware
• The entire point of pixel art is crisp discrete colors, not neural approximation
The insight NTC gives us — the style latent concept
NTC's core idea: encode correlated visual information as a compact neural representation, then decode on demand.

For 3D: albedo and normal are correlated (detail in one corresponds to detail in the other) → compress together.

For pixel art: tiles in a set are correlated — they share lighting direction, outline weight, pixel density, dithering patterns, palette relationships.

The adaptation: encode the STYLE of a tileset as a compact "style token" that conditions every new generation. Not a neural decoder — a statistical fingerprint.
PAX Style Latent — NTC's idea applied to pixel art generation
When an artist creates their first 5–10 tiles, PAX's style encoder analyzes them and extracts a compact style token. This token is stored in the .pax project and injected into every subsequent generation prompt. New tiles are generated consistent with the established style — without the artist having to re-specify every constraint each time.

Outline styleSelf-outline weight, selective, drop-shadow, or none

Hue biasWarm or cool, saturation envelope
Symmetry tendencyWhat % of tiles use horizontal/vertical symmetry
What this enables that nobody has yet: An indie developer can draw 8 tiles by hand — imperfect, rough, but with their own style — and PAX extracts the style fingerprint from those 8 tiles. Every subsequent generated tile comes out looking like it belongs with those 8. The system learns the human's style from minimal examples and enforces it on AI-generated output. This is the NTC insight applied to authorship rather than compression. It's a completely novel feature no pixel art tool has.
