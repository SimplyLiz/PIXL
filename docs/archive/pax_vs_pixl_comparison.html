What neither spec has — the real next level
1. Vision feedback loop — the biggest missing insight
Modern Claude has vision. After rendering a tile to PNG, the MCP tool can return it as base64 and the LLM can actually see what it drew and self-correct. Neither spec exploits this. This changes everything: the LLM is no longer writing blind. Error correction goes from "parse validation errors" to "I can see the edge doesn't look right." This is a completely different quality ceiling.
2. Semantic tile DNA — beyond edge classes
Edge classes only check geometric compatibility. Semantic tags ("wall", "floor", "transition") are defined in PAX but unused by WFC. The next level: WFC uses semantic rules — "a wall tile can never be directly above a water tile", "moss only appears adjacent to floor tiles". This produces game-sensible maps, not just geometrically valid ones.
3. Progressive refinement passes
Instead of one-shot generation, the workflow is: (1) rough structural pass — LLM places semantic blocks, (2) detail pass — AI-assisted fill within blocks using stamps, (3) variation pass — introduce cracks, moss, variation without breaking edges. Each pass is smaller and more reliable than one monolithic generation attempt.
4. Stamp corpus as community asset
A curated, versioned library of named stamps (corner_stone, crack_diagonal, moss_patch, torch_bracket…) shared across projects. LLMs reference the corpus by name. Over time, the corpus grows via contribution — it becomes a visual vocabulary that gets richer with every project, making generation more reliable on every new tileset.
5. Symmetry-first generation for large sprites
PIXL mentions symmetry declarations but doesn't go far enough. The key insight: a 32×32 sprite with 4-way symmetry only requires an 8×8 quadrant — well within LLM reliability. The tool should automatically detect symmetry opportunities from the tile type (a gem, a coin, a circular motif) and request only the minimal unique region from the LLM.
Verdict
PAX wins on format and algorithms. PIXL wins on UX and vision. Neither is complete alone. PAX's TOML + three-tier encoding + fuzzy edge classes + concrete WFC spec is the right technical foundation. PIXL's symmetry system + @theme + BPE framing + PIXL Studio integration is the right product vision. The merged spec — call it PAX 2.0 — takes PAX's bones and grafts PIXL's soul onto them. Then the vision feedback loop, semantic tile DNA, and progressive refinement are what make it genuinely groundbreaking.
