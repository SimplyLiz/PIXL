# Pixel Art Scaling and Upscaling Algorithms

## Overview

Pixel art scaling algorithms are graphical filters designed to enhance hand-drawn 2D pixel art graphics. Unlike general image scaling (bilinear, bicubic), these algorithms preserve sharp edges, avoid blurring, and respect the integer-pixel nature of the source art.

## Nearest Neighbor

The simplest approach — each pixel is duplicated N×N times. Preserves exact colors and sharp edges but creates visible "blockiness." Used as baseline and when exact pixel reproduction is desired.

## EPX (Eric's Pixel Expansion) / Scale2×

**Origin**: Developed by Eric Johnston at LucasArts (~1992) for porting SCUMM games from PC to Macintosh.

**Algorithm** (2× scale): For each source pixel P with neighbors A (up), B (right), C (left), D (down):

```
If C == A and C != D and A != B → output[top-left] = A
If A == B and A != C and B != D → output[top-right] = B
If D == C and D != B and C != A → output[bottom-left] = C
If B == D and B != A and D != C → output[bottom-right] = D
Otherwise → output = P (copy original)
```

**Effect**: Smooths diagonal lines and curves while preserving right angles. Only uses colors already present in the original image.

**Scale3×** extends this to 3× by examining more neighbors.

## Eagle

**Algorithm** (2× scale): Examines all 8 surrounding pixels. For each quadrant of the output:
1. Start with the center pixel
2. If the three nearest neighbors all match, use their color instead

**Weakness**: Creates unwanted single-pixel artifacts when surrounded by identical colors.

## hqx Family (hq2x, hq3x, hq4x)

**Origin**: Maxim Stepin, 2003. Default in MAME arcade emulator.

**How it works**: Uses large lookup tables generated by pattern analysis:
1. Compare center pixel to all 8 neighbors using configurable threshold
2. This creates a pattern key (up to 2^8 = 256 base patterns)
3. Each pattern maps to a specific blend recipe for sub-pixels
4. **Introduces interpolated colors** not in the original palette

**Effect**: Very smooth curves and diagonals. Can look "soft" compared to the original pixel art style. Each scaling factor (2×, 3×, 4×) has its own lookup table.

## xBR / xBRZ

**Improvement over hqx**: Uses pattern matching like hqx but with:
- Edge detection that considers more context
- Better handling of small features and thin lines
- **xBRZ**: Optimized for multi-core CPUs, 40–60% faster than hqx
- Supports alpha channel
- Scaling factors from 2× to 6×

## RotSprite

**Origin**: Xenowhirl, 2007.

**Unique feature**: Combines scaling with arbitrary-angle rotation. The algorithm:
1. Scale image 8× using Scale2× (applied 3 times)
2. Rotate the enlarged image using nearest-neighbor
3. Scale back down to desired size

This produces dramatically cleaner rotations than rotating the original pixels directly.

## Kopf-Lischinski Depixelizing

**Origin**: Johannes Kopf and Dani Lischinski, 2011 (SIGGRAPH paper).

**Fundamentally different approach**: Converts pixel art to resolution-independent vector graphics.

**Algorithm**:
1. Build a similarity graph connecting adjacent pixels
2. Resolve ambiguities at junctions using heuristics (prefer long curves over short ones)
3. Extract B-spline curves from the resolved graph
4. Fill regions with smooth color gradients

**Effect**: Produces infinitely scalable vector art that preserves the artist's intent. Sharp edges remain sharp; curves become smooth mathematical curves.

**Limitation**: Works best on character sprites and simple scenes. Complex textures or dithering patterns can confuse the similarity graph.

## Comparison Summary

| Algorithm | Scale | New Colors? | Speed | Best For |
|-----------|-------|-------------|-------|----------|
| Nearest Neighbor | Any | No | Fastest | Exact reproduction |
| EPX/Scale2× | 2×/3× | No | Fast | Clean upscaling, preserving palette |
| Eagle | 2× | No | Fast | Simple sprites |
| hq2x–hq4x | 2×–4× | Yes | Medium | Emulators, smooth look |
| xBRZ | 2×–6× | Yes | Fast | Modern emulators |
| RotSprite | Any + rotation | No | Slow | Rotation without artifacts |
| Kopf-Lischinski | Infinite | N/A (vector) | Slow | Print, high-res display |
